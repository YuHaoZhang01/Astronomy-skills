# Astrophysics Paper Data Finder

## 概述

- 先满足科学筛选条件，再满足公开数据条件。
- 只接受论文明确公开、或论文直接指向可信科学档案的数据。
- 原始数据必须原样保存，不覆盖、不改名、不转码替换原件。
- 每次任务都要产出可复查的目录：检索记录、参考文献、原始数据、数据清单、Jupyter 核对 notebook、核对说明。
- 遇到“近五年”“最近十篇”“今年以来”这类相对时间请求时，必须实时联网检索，并在结果中写出明确绝对日期范围。

## 依赖条件

- 允许联网，用于检索 ADS、期刊页面、arXiv 和公开数据档案。
- 允许写本地文件，用于保存原始数据和运行产物。
- 若需要生成 notebook，系统中应有可运行的 Python 解释器；生成 notebook 只依赖标准库，实际运行 notebook 时通常还需要 `pandas`、`matplotlib`，以及读取 ECSV 或 FITS 时需要 `astropy`。

## 工作原则

- 优先保真，不优先凑数。
- 明确区分“找到相关论文”和“成功获得公开数据”。
- 若公开数据不足用户要求数量，如实说明缺口，不得用未公开数据、图像数字化结果或来源不明的二次整理包补齐。
- 除非用户明确要求，否则不要从论文图像里手动抠点。
- 所有结论都要能回溯到具体论文、具体链接、具体文件和具体列名。

## 标准工作流

### 1. 解析请求

- 提取主题、时间范围、目标数量、数据类型和硬性排除条件。
- 识别主题同义词、缩写和拼写变体。
- 若用户没有指定数据类型，优先寻找最直接支撑论文结论的公开机器可读数据。
- 若用户没有指定目标数量，把范围做全，但最终输出按质量排序。

### 2. 规范时间范围

- 把相对时间转换成绝对日期。
- 在检索记录中写出执行日期和转换结果。
- 例如在 2026-03-11 执行“近五年”时，应解释为 `2021-03-11` 到 `2026-03-11`。

### 3. 实时检索论文

- 优先使用 NASA ADS 做主题检索和书目信息归集。
- 继续打开论文正式页面、arXiv 页面、出版社补充材料页和数据可得性声明页核实数据是否公开。
- 将主题词、数据类型词和公开数据词组合检索，例如 `light curve`、`photometry`、`machine-readable table`、`supplementary data`、`VizieR`、`Zenodo`。
- 需要选择站点和下载源时，读取 [references/data-sources.md](references/data-sources.md)。

### 4. 过滤候选论文

- 仅保留同时满足以下条件的论文：
- 主题相关。
- 时间范围符合要求。
- 数据类型与用户请求匹配。
- 存在明确可访问的公开下载入口。
- 按以下优先级选择数据来源：
- 论文补充材料中的机器可读表。
- CDS 或 VizieR 的机器可读表。
- 论文直接链接的 Zenodo、Figshare、Dataverse 或项目仓库。
- MAST、HEASARC、IRSA、ESO 等官方科学档案中由论文明确指向的数据。
- 明确排除以下情况：
- 只有图片、没有原始数据文件。
- 标注“可按请求提供”但没有公开下载地址。
- 来源不明的镜像站、论坛贴子、博客转载或第三方汇总页。

### 5. 建立任务目录

- 为每次请求创建独立运行目录。
- 将原始请求、绝对日期范围、检索词和假设写入 `request.json`。
- 将检索过程、淘汰原因和最终入选依据写入 `search-log.md`。
- 目录结构和 `datasets.json` 规范见 [references/output-layout.md](references/output-layout.md)。

### 6. 保存参考文献

- 为每篇入选论文记录标题、作者、年份、期刊、DOI、ADS 或 arXiv 链接、数据链接和入选理由。
- 输出两个文件：
- `references/references.md`，用于人工审查。
- `references/bibliography.bib`，用于后续引用和写作。
- 若 ADS 或出版社提供 BibTeX，优先保存其原始条目。

### 7. 下载并保存原始数据

- 将每篇论文的数据保存到 `data/<paper-slug>/original/`。
- 为作图或解析产生的解压文件、副本或清洗件放到 `data/<paper-slug>/working/`。
- 保留原始压缩包、原始文件名和原始扩展名。
- 不要将 CSV 转成 XLSX 替换原文件，也不要将 FITS 转成 CSV 后丢弃原件。
- 为每个文件记录来源 URL、下载日期、文件格式和用途。
- 若链接失效、返回 404、需要登录或无法确认与论文一一对应，标记为“不满足公开下载条件”。

### 8. 生成数据清单

- 在 `manifests/datasets.json` 中为每个可作图的数据文件写一条记录。
- 明确填写作图所需列名，不要依赖脚本猜测。
- 对光变曲线优先填写以下信息：
- 时间列及其系统，例如 `JD`、`MJD`、`phase`。
- 波段列，例如 `filter`、`band`、`instrument`。
- 亮度列，例如 `mag`、`flux`、`luminosity`。
- 误差列、上限标记、测光系统和必要的单位说明。

### 9. 生成 Jupyter 核对 notebook

- 运行 `scripts/make_review_notebook.py` 从 `datasets.json` 自动生成 notebook。
- 让 notebook 在原始数据已经下载完成后可离线重跑。
- 在 notebook 或 `datasets.json` 的注释中写清楚影响复现的转换，例如：
- `JD - 2400000.5 -> MJD`。
- 星等轴反向显示。
- 仅绘制某些波段。
- 删除空值、上限点或明显非测光列。

### 10. 复画并核对

- 运行 notebook 后，将核对结果写到 `qa/plot-check.md`。
- 至少说明：
- 哪些图已经与原文趋势一致。
- 哪些图仍有差异。
- 差异来自列选择、单位换算、相位定义、上限处理还是原始数据缺失。
- 若最终公开数据不足目标数量，明确写出“成功获得公开数据的论文数”和“缺口原因”。

## 交付标准

- 最终答复必须给出精确数量。
- 明确区分“相关论文数”和“已成功下载公开数据的论文数”。
- 所有原始文件都要在本地可定位。
- 所有图都要能追溯到具体文件路径、列名和论文。
- 若做过任何清洗或格式转换，必须保留原件并单独说明。

## 失败处理

- 若主题过窄导致公开数据不足，先扩大检索广度，再维持同样严格的数据公开标准。
- 若同一论文的数据分散在多个站点，优先保留论文主链接对应的数据版本，并在参考文献中记录备用链接。
- 若无法确认某份数据是否与论文直接对应，不将其记入最终“成功下载”清单。

## 资源导航

- 判断哪些站点可信、哪些数据可接受时，读取 [references/data-sources.md](references/data-sources.md)。
- 创建运行目录、填写 `datasets.json` 和组织交付物时，读取 [references/output-layout.md](references/output-layout.md)。
- 生成 notebook 时，运行 [scripts/make_review_notebook.py](scripts/make_review_notebook.py)；只在必要时手改输出的 `.ipynb`。

## 示例触发语句

```text
帮我寻找近五年来，10 个关于 Ibn 型超新星的光变曲线数据。
```

```text
查找过去三年 tidal disruption event 的公开光谱数据，保留原始 FITS 和引用信息。
```

```text
帮我整理 20 篇关于快速蓝光学瞬变体的公开测光表，并生成一个 notebook 复画论文里的主图。
```
